# 示例：从论文和代码提炼专利点

## 输入材料

- 一篇方法论文 PDF，包含模型结构、损失函数和实验设置。
- 一个代码仓库，包含模型定义、训练入口、推理脚本和配置文件。
- 可选：已有专利交底书模板或相关现有专利。

## 推荐提问

```text
使用 $chinese-patent-disclosure，阅读这篇论文和代码，先扫描项目并总结 3-5 个候选专利点，判断哪些点具备与现有技术的区分度。确认后按我的交底书模板撰写完整发明专利交底书。
```

## 预期输出

- 项目扫描摘要：技术主题、输入输出、核心模块、训练/推理流程。
- 候选专利点：每个点说明技术问题、技术机制、材料依据、区分度和是否保留。
- 交底书正文：背景技术、现有技术缺点、技术方案、关键点与保护点、替代方案、技术优点。
- 自检意见：公式变量、逻辑闭环、前后文一致性和专利风格问题。

## 局部审查示例

```text
使用 $chinese-patent-disclosure，只检查这份 DOCX 中公式变量解释是否完整，列出问题和修改建议，不要直接改文件。
```

```text
使用 $chinese-patent-disclosure，基于现有交底书模板重写关键点和欲保护点，要求一一对应，并用高亮标识新增内容。
```
