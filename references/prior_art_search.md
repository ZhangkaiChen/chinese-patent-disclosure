# 查新与现有技术检索规范

## 适用时机

正式成稿、重写“现有技术”、撰写“与最接近现有技术相比的技术优点”或判断专利点区分度时，执行轻量查新。快速润色、局部公式检查、附图调整、小范围语言优化时不强制查新，除非用户要求。

## 检索优先级

1. 优先使用国知局中国专利公布公告检索：`http://epub.cnipa.gov.cn/`。
2. 若当前 skill 中存在 `scripts/cnipa_epub_search.py`，优先使用该脚本辅助检索；该脚本依赖同目录的 `cnipa_epub_parse.py`、`cnipa_epub_crawler.py` 和 `requirements-cnipa.txt`。
3. 国知局检索异常、无结果、关键词不稳定或需要论文对比时，降级使用 WebSearch、Google Scholar、Google Patents 或用户提供的论文/专利。
4. 对用户明确提供的现有专利，应优先阅读全文或摘要、权利要求、说明书关键段落，不只读标题。

## CNIPA 脚本使用规则

使用脚本前，必要时安装依赖：

```bash
pip install -r scripts/requirements-cnipa.txt
python -m playwright install chromium
```

检索词由 Agent 根据本案技术方案归纳 2-8 个高相关语义块，例如专业术语、名词短语、名动组合或业内固定搭配。不要把一整句长中文作为唯一参数，也不要用“系统”“方法”“增强”等泛词单独检索。

执行时遵守：

- 每次命令只传一个检索单位，进行多轮调用；
- 每轮读取 stdout 中唯一一行 `EPUB_HITS_JSON:` 后的 JSON 数组；
- 多轮结果由 Agent 按 `pub_number` 去重合并；无 `pub_number` 时按 `link`，再按标题前缀去重；
- stderr 中的 `EPUB_NOTE`、`EPUB_HINT`、`EPUB_MERGE` 不是失败依据，解析以 stdout 的 JSON 为准；
- 命令非 0、超时、无 Playwright、JSON 为空数组或结果明显无关时，降级 WebSearch。

示例：

```bash
python scripts/cnipa_epub_search.py 缺模态
python scripts/cnipa_epub_search.py 交叉注意力
python scripts/cnipa_epub_search.py 遥感多模态
```

## 摘要使用规则

若脚本结果中某条包含非空 `abstract`：

- 写入查新笔记或现有技术章节前，必须先理解该摘要中的技术领域、解决问题、核心手段和技术效果；
- 正文不得大段复制官方摘要，应压缩为“方案概括 + 应用场景 + 局限性”；
- 若标题和摘要理解不一致，以摘要为准；
- 若无 `abstract`，应通过国知局详情页、Google Patents 或其他可核验来源补全理解，不得仅凭标题臆测。

## 检索关键词

关键词应覆盖：

- 应用场景，例如遥感、多模态、作物识别、地物分类、目标检测；
- 核心机制，例如缺模态、有效性掩码、质量评分、交叉注意力、Token 对齐、动态推理；
- 数据类型，例如光学、SAR、高分影像、时序影像；
- 任务输出，例如分类、分割、检测、质量评价；
- 论文或代码中的专有模块名和可替代表述。

检索时先用宽关键词找到相近领域，再用核心机制缩小范围。

## 结果记录

查新结果可写入工作材料或用于正文第一章现有技术。记录内容包括：

- 文献或专利名称；
- 公开号/论文题名/链接；
- 技术路线摘要；
- 与本申请的相同点；
- 未覆盖的问题；
- 本申请的差异化机制。

完整检索过程、失败记录和无关结果不写入正文。不得虚构专利公开号、论文结论或检索结果。

## 正文写法

正文中只保留与本申请最接近、最能说明差异的现有技术。可使用：

“公开文献/专利……公开了……，其主要通过……实现……。该方案能够……，但其技术重点在于……，未针对……形成……。本申请相对于该方案，在……阶段引入……，从而……。”

正文第一章可写检索说明，但只能写公开数据库、公开渠道和主要检索词。例如：

“检索说明：在国家知识产权局专利公布公告系统及 Google Patents 中，以……为检索词进行检索；部分条目的公开文本与著录项以 Google Patents 页面复核。”

不要在正文写入脚本名、Playwright、WebSearch 降级、Agent、skill 仓库等内部流程信息。没有确定公开号或出处时，不要写成具体专利，只能写成现有技术类别或公开论文类别。
