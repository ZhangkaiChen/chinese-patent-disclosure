# 现有技术与创新点判断规范

## 基本原则

撰写现有技术和创新点时必须先读用户提供的论文、代码、专利或模板。不能仅凭题名推断技术内容。

现有专利不是必需材料。若用户未提供现有专利，可以根据论文引用、公开论文、代码实现和通用技术路线总结背景技术，但不得虚构具体专利公开号、发明名称或公开内容。若用户提供现有专利，则优先把其作为最接近现有技术或模板来源进行分析。

## 论文/代码中的创新点提炼

从论文或代码中提炼专利点时，逐项判断：

1. 是否解决了明确技术问题，而不是单纯提高指标；
2. 是否具有具体技术手段，例如网络模块、数据流、掩码机制、门控机制、损失约束、动态推理规则、质量评价输出等；
3. 是否能与现有技术形成区分，而不是已有论文主贡献的简单复述；
4. 是否有源材料支撑，包括论文公式、结构图、消融实验、代码模块或参数配置；
5. 是否能写成“关键点”和“欲保护点”；
6. 是否可以在替代骨干网络或不同数据集下仍保持技术思想成立。

优先保护具体机制，而不是宽泛概念。例如，与其写“多模态融合模型”，不如写“在注意力归一化前利用有效性掩码屏蔽无效连接，并结合质量门控调节模态贡献的融合机制”。

## 常见现有技术类别

遥感、多模态和深度学习专利中常见现有技术包括：

- 多模态特征融合：数据拼接、特征拼接、双分支网络、交叉注意力、门控融合；
- 跨分辨率或多尺度统一编码：连续尺度特征、尺度自适应 patch 编码、多传感器统一模型；
- 多模态自监督表征：跨模态对比、重建、Token 对齐、统一预训练；
- 缺失模态处理：补零、均值填充、插值、生成式补全、模态 dropout、按组合重训；
- 多任务适配：模型库选择、混合专家路由、任务头、超参数寻优；
- 样本生成与数据增强：扩散模型、生成对抗网络、伪标签扩增、条件生成。

## OmniSat / AnySat 等论文对比方式

涉及 OmniSat、AnySat 或类似基础模型论文时，应谨慎表述：

- OmniSat 可作为多模态自监督融合、模态专属编码、跨模态 Token 对齐和重建目标的现有技术基础；
- AnySat 可作为多分辨率、多尺度、多传感器统一编码和预训练的现有技术基础；
- 本申请不宜主张“首次提出多模态统一表征”或“首次提出跨模态 Token 对齐”；
- 可主张的差异通常应落在面向实际部署的完整闭环，例如质量评分、多级有效性掩码、跨分辨率 Token 属性绑定、质量驱动缺模态训练、Softmax 前掩码注意力、质量门控融合、完整/缺失视图一致性、任意缺模态动态推理和质量评价输出。

## 现有专利模板处理

当用户提供现有专利或交底书模板时：

1. 先提取模板的章节顺序、标题格式、段落长度、公式说明方式、关键点/保护点写法和附图说明方式；
2. 再把当前论文或代码的技术内容填入模板，不改变模板的整体格式；
3. 模板中的旧技术内容不能机械保留，必须替换为当前发明的真实模块和任务范围；
4. 如果模板与当前发明不完全匹配，应保留格式，调整内容；
5. 不得把模板中的旧模态、旧任务或旧公式带入新专利。

## 对比段落写法

可使用以下句式：

“公开文献/专利……公开了……，其主要通过……实现……。该方案能够……，但其技术重点在于……，通常假设……，未针对……形成……闭环。本申请相对于该方案，在……阶段引入……，在……阶段执行……，在……阶段输出……，从而……。”

没有具体专利号时，不要虚构编号，可写“现有多模态融合方法通常……”“以……论文为代表的方案……”。

## 有效创新点判断

最终进入关键点和保护点的内容应满足：

- 是技术机制，而不是业务目标或数据结果；
- 有明确输入、处理、输出和技术效果；
- 能解决背景技术中的一个或多个缺陷；
- 与最接近现有技术相比有结构、流程或约束方式差异；
- 在不同实施例中可替换部分组件但核心思想不变；
- 能在权利要求中被拆成步骤、模块或装置单元。

## 较强的可保护对象

通常比“使用 AI 模型”更有价值的保护对象包括：

- 影响模型运行的数据组织或样本构造机制；
- 网络模块之间的输入输出关系、门控关系、掩码关系或注意力关系；
- 具有正负样本、教师学生关系或任务权重的损失函数；
- 与质量阈值、有效性掩码、可用模态或任务类型绑定的动态推理路径；
- 面向业务复核的置信度图层、质量评价图层或低可信区域标记；
- 可替换骨干网络但保持核心机制不变的架构设计。
